Introduction to bAIsed

bAIsed is a high-precision fairness auditing framework for modern AI models. It provides developers with the toolset needed to detect, measure, and mitigate algorithmic bias throughout the entire machine learning lifecycle.

tune

Precision Control

Audit your models with granular control over sensitive attributes and intersectional parity checks.

waves

Real-time Drift

Monitor statistical parity in flight with live alerts when model performance begins to skew across cohorts.

shield

Compliance Ready

Generate automated fairness reports designed to meet strict algorithmic accountability regulations.

Quick Start

Get started by installing the bAIsed core CLI. Ensure you have Node.js 16+ installed in your development environment.

terminal
1 npm install @baised/core
2 baised init --project my-model-audit

Metrics Engine Section

Our proprietary fairness engine calculates the Disparate Impact Ratio (DIR) across multidimensional vectors using a modified Four-Fifths rule algorithm.

DIR = P(Y=1 | D=unprivileged) / P(Y=1 | D=privileged)

Authentication

bAIsed uses API keys to authenticate requests. You can view and manage your API keys in the settings dashboard. Include your API key in the `Authorization` header as a Bearer token.

Authorization: Bearer sk_test_12345

Endpoints

The primary endpoint for fairness analysis is /analyze. The API expects a JSON payload containing dataset metadata and model predictions.

Rate Limits

Free tier users are limited to 100 requests per minute. Pro and Enterprise users have unlimited API access.

Bias Detection

Our bias detection algorithm automatically scans features for correlation with protected classes using Disparate Impact and Equal Opportunity metrics.

Drift Analytics

Monitor real-time changes in dataset distribution and parity metrics over time with continuous integration.

Fairness Ratios

bAIsed provides a suite of fairness ratios including Statistical Parity Difference, Average Odds Difference, and Predictive Equality.
